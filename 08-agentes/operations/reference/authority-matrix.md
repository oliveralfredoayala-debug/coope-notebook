# Canonical authority matrix

| Concern | Canonical owner | Producers may do | Others must not do |
|---|---|---|---|
| Pipeline order | Operations | declare prerequisites | reorder the factory independently |
| Content classification/catalog | Coordinator | provide metadata | create competing IDs/catalog rules |
| Artifact text and pedagogy | Artifact Agents 1–2 | author and revise | rewrite during design/QA |
| Artifact behavior, bars, numbers, calculations | Artifact Agent 3 | implement approved plan | simplify or redesign functionality |
| Legal meaning/source | LexCoop or declared legal authority | cite grounded source | infer or silently alter |
| Verbatim legal presentation | Legal Verbatim Guardian + Palette Architect | wrap exact text | paraphrase inside legal block |
| Orthography and language | Language Integrity Auditor | correct form outside verbatim text | change legal/pedagogical meaning |
| Global top bar, left tabs renderer, right artifact drawer, footer | Global Navigation | submit tabs as data | render competing global menus |
| Semantic color | Palette Architect | consume tokens | invent semantic colors |
| Capsule internal visual resources | Capsule Visual System | compose approved components | override global shell |
| Team activity mechanics | Team-work orchestrator | define activity | navigation/color agents changing mechanics |
| P2L game rules | P2L orchestrator | define game | global agents changing rules |
| Repository placement | Content Librarian / Antigravity | provide package | agents inventing repo paths |
| Final compliance decision | Delivery Manager | submit evidence | self-approve |

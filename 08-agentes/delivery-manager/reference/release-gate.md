# Release gate

A release cannot pass until the following evidence is present:

| Gate | Evidence |
|---|---|
| User intent | exact request and exclusions |
| Scope | content type and protected zones |
| Authority | active-agent matrix |
| Sequence | Operations ledger |
| Content | applicable domain audit |
| Legal | source/verbatim audit when applicable |
| Language | orthography/character integrity audit |
| Design | visual-system compliance |
| Color | semantic palette audit |
| Navigation | global shell non-competition audit |
| Function | applicable functional QA |
| Accessibility | keyboard, labels, focus, responsive evidence |
| Repository | paths, manifests, handoff |
| Final | Delivery Manager decision |

Any “not applicable” entry must include a reason.
